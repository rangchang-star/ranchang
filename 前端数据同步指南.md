# 同步数据到前端 - 操作步骤

## 📋 概述

前端页面已经修改为从 Supabase 数据库获取用户数据，无需手动同步！

## 🚀 操作步骤

### 第 1 步：在 Supabase 控制台创建表

1. 登录 [Supabase 控制台](https://supabase.com/dashboard)
2. 选择你的项目（`yqttwnpuhkfytrnslqml`）
3. 点击左侧菜单的 **SQL Editor**
4. 点击 **New query**
5. 复制下面的 SQL，点击 **RUN**

```sql
-- 创建 users 表
CREATE TABLE IF NOT EXISTS users (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(128) NOT NULL,
  age INTEGER,
  avatar TEXT,
  phone VARCHAR(20),
  email VARCHAR(255),
  connection_type VARCHAR(50),
  industry VARCHAR(100),
  need TEXT,
  ability_tags JSONB,
  resource_tags JSONB,
  level VARCHAR(50),
  company VARCHAR(255),
  position VARCHAR(255),
  status VARCHAR(20) DEFAULT 'active',
  is_featured BOOLEAN DEFAULT false,
  join_date TIMESTAMP WITH TIME ZONE,
  last_login TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE
);

-- 创建索引
CREATE INDEX IF NOT EXISTS users_connection_type_idx ON users(connection_type);
CREATE INDEX IF NOT EXISTS users_email_idx ON users(email);
CREATE INDEX IF NOT EXISTS users_industry_idx ON users(industry);
CREATE UNIQUE INDEX IF NOT EXISTS users_email_unique ON users(email);
```

## 🆘 故障排查

### 问题：无法连接数据库

**解决方案**：
- 检查 `.env.local` 文件中的 Supabase 配置
- 确认 Supabase 项目处于活跃状态

### 问题：表不存在

**解决方案**：
- 确保在 Supabase 控制台执行了创建表的 SQL
- 检查 SQL 是否执行成功（查看绿色成功提示）

### 问题：前端无法显示数据

**解决方案**：
- 检查浏览器控制台是否有错误
- 刷新页面
- 检查 Supabase API 权限设置
